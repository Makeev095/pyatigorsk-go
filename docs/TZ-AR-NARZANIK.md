# Техническое задание: AR-режим «Нарзанник» для Pyatigorsk GO

## 1. Контекст и аналогия с Pokémon GO

### Как устроена Pokémon GO (релевантные механики)

1. **Геолокация (GPS)** — определяет позицию пользователя с точностью 5–10 м.
2. **Камера** — даёт видеопоток окружения, на который накладывается AR-контент.
3. **Гироскоп и акселерометр** — определяют направление взгляда и ориентацию устройства, чтобы персонаж не «уплывал» при повороте.
4. **AR-режим** — при нахождении на точке (PokéStop/Gym) игрок может включить камеру и увидеть персонажа в AR, сфотографировать его (Snapshot).
5. **Niantic VPS** (Visual Positioning System) — для точного привязывания AR к миру (сантиметровая точность). В нашем случае **не требуется**: достаточно GPS + компас, как в базовом режиме Pokémon GO.

### Целевая механика для Pyatigorsk GO

- Пользователь **находится в радиусе достопримечательности** (как сейчас).
- Вместо простого нажатия «Открыть» — **открывается камера** с AR-режимом.
- В камере отображается **анимированный персонаж «Нарзанник»** (связан с нарзаном, минеральной водой Пятигорска).
- Пользователь **фотографирует** персонажа в камере.
- После успешного снимка точка **отмечается как пройденная** (discovered), начисляется XP.

## 2. Текущее состояние проекта Pyatigorsk GO

### Структура данных

- **Landmark** (`src/types.ts`): `id`, `name`, `description`, `lat`, `lon`, `radiusM`, `xp`, `source?`
- **PlayerProgress**: `discovered: Record<LandmarkId, { discoveredAt }>`, `xp`
- **Хранение**: `localStorage` через `src/lib/storage.ts`

### Логика обнаружения

- `scanNearby()` и кнопка «Открыть» — проверка `distM <= lm.radiusM + accuracyBoostM`
- `discover(lm)` — добавляет в `progress.discovered`, начисляет XP

### Что отсутствует

- Камера (`getUserMedia`, `MediaStream`)
- AR-библиотеки (WebXR, AR.js, Three.js и т.п.)
- Персонажи, анимации, спрайты

## 3. Функциональные требования

### 3.1. Условие входа в AR-режим

- Пользователь **в радиусе** достопримечательности: `distM <= lm.radiusM + accuracyBoostM`
- Геолокация активна (`geo.status === 'watching'`)
- Точка **ещё не пройдена** (`!discoveredIds.has(lm.id)`)

### 3.2. Переход в AR-режим

- Вместо/дополнительно к кнопке «Открыть» — кнопка **«Найти Нарзанника»** (или аналогичная).
- По нажатию:
  1. Запрос разрешения на камеру (`navigator.mediaDevices.getUserMedia({ video: true })`)
  2. Открытие полноэкранного AR-экрана с видеопотоком камеры
  3. Наложение персонажа «Нарзанник» поверх видео

### 3.3. AR-экран

- **Видео с камеры** — фон (fullscreen или основной слой).
- **Персонаж «Нарзанник»**:
  - Анимированный (idle, возможно лёгкие движения)
  - Связан с темой нарзана (минеральная вода Пятигорска): бутылка, капля, курортный персонаж и т.п.
  - Позиционирование: центр экрана или фиксированная область (как в Pokémon GO — персонаж «стоит» перед камерой)
  - Учёт ориентации устройства (DeviceOrientationEvent) — персонаж остаётся стабильным при повороте телефона

### 3.4. Фотографирование

- Кнопка **«Сфотографировать»** (или съёмка по тапу).
- Действия:
  1. Захват кадра с видео + персонажем (Canvas API или `captureStream` + `drawImage`)
  2. Сохранение снимка (опционально — в галерею или только в приложении)
  3. Вызов `discover(lm)` — точка отмечается как пройденная, начисляется XP
  4. Показ toast: «Нарзанник пойман! Открыто: [название] (+N XP)»
  5. Закрытие AR-экрана, возврат к карте

### 3.5. Выход без фото

- Кнопка «Назад» / «Отмена» — закрыть AR без `discover`, вернуться к карте.
- Точка остаётся непройденной.

## 4. Технические рекомендации

### 4.1. Варианты реализации AR

| Вариант | Плюсы | Минусы |
|--------|-------|--------|
| **WebXR + Three.js** | Настоящий AR, hit-test, поддержка ARCore | Сложнее, не все браузеры |
| **Canvas overlay** | Просто, работает везде | Персонаж «плавает» на экране, без привязки к миру |
| **model-viewer + WebXR** | Готовый компонент для 3D | Нужна 3D-модель |

**Рекомендация для MVP**: Canvas overlay — видео с камеры + Canvas поверх с отрисовкой 2D-спрайта/анимации Нарзанника. Минимальные зависимости, быстрая реализация.

### 4.2. Персонаж «Нарзанник»

- **Формат**: серия PNG-спрайтов (sprite sheet) или Lottie-анимация, или простой SVG/CSS-анимация.
- **Тема**: нарзан (минеральная вода), курорт, Пятигорск — дружелюбный персонаж (капля, бутылка с лицом, курортник и т.п.).
- **Размер**: адаптивный, ~30–40% ширины экрана.
- **Позиция**: нижняя треть экрана (как в Pokémon GO — «стоит на земле» перед камерой).

### 4.3. Захват фото

```ts
// Псевдокод
const video = document.querySelector('video')
const canvas = document.createElement('canvas')
canvas.width = video.videoWidth
canvas.height = video.videoHeight
const ctx = canvas.getContext('2d')
ctx.drawImage(video, 0, 0)
// + отрисовка персонажа поверх, если он рисуется отдельно
const blob = await new Promise<Blob>(cb => canvas.toBlob(cb, 'image/jpeg', 0.9))
// blob можно сохранить или отправить
```

### 4.4. Интеграция с App.tsx

- Новое состояние: `arMode: { landmark: Landmark } | null`
- При `arMode !== null` — рендерить AR-компонент поверх всего (fullscreen overlay).
- Кнопка «Найти Нарзанника» вместо/рядом с «Открыть» — только когда `inRange && !discovered`.

## 6. UX-поток (пошагово)

1. Пользователь на карте, выбирает достопримечательность, подходит в радиус.
2. Видит кнопку **«Найти Нарзанника»** (или «Открыть в AR»).
3. Нажимает → запрос доступа к камере.
4. Открывается AR-экран: камера + Нарзанник.
5. Пользователь наводит камеру, при желании поворачивает телефон.
6. Нажимает **«Сфотографировать»**.
7. Снимок делается → `discover(lm)` → toast → закрытие AR, возврат к карте.
8. Точка отмечена как пройденная (зелёная на карте), XP начислен.

---

## 7. Ограничения и fallback

- **Нет камеры / отказ в доступе**: показать toast «Для AR нужна камера», оставить обычную кнопку «Открыть» (текущая логика).
- **Браузер без `getUserMedia`**: fallback на «Открыть» без AR.
- **HTTPS**: камера в браузере требует HTTPS (для PWA — обычно уже есть).

## 8. Критерии приёмки

- [ ] При нахождении в радиусе непройденной точки доступна кнопка «Найти Нарзанника».
- [ ] По нажатию открывается камера с AR-режимом.
- [ ] В камере виден анимированный персонаж Нарзанник.
- [ ] Кнопка «Сфотографировать» делает снимок и отмечает точку как пройденную.
- [ ] После съёмки начисляется XP, показывается toast, происходит возврат к карте.
- [ ] Кнопка «Назад» закрывает AR без отметки точки.
- [ ] При отказе в камере доступна обычная кнопка «Открыть».
